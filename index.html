✅ 프로젝트 요약

👨‍💻 내가 기여한 부분

🚨 겪은 트러블 슈팅 및 해결과정

1. 자판기와의 양방향 WebSocket 통신 구현

  • 문제 상황 : 자판기에서 주문에 대한 응답을 서버가 처리하여 사용자에게 보내야하는 요구사항이 있었습니다. 하지만 기존의 REST API 통신으로는 사용자와 서버 간 실시간 양방향 통신이 불가능했습니다.
  • 해결 방안 : STOMP(Simple Text Oriented Messaging Protocol) 형식의 WebSocket을 도입하여 양방향 통신을 구현했습니다. 이를 통해 자판기의 주문 처리 결과를 서버가 실시간으로 받아 사용자에게 전달할 수 있게 되었습니다.

2. Security Filter 단에서 발생한 예외처리 핸들링

  • 문제 상황 : Spring Security의 Filter 체인에서 발생하는 예외는 DispatcherServlet에 도달하기 전에 발생하므로, 전역 핸들러로 처리할 수 없었습니다. 이로 인해 인증 실패, 권한 부족 등의 보안 관련 예외에 대해 일관된 에러 응답을 클라이언트에 제공하기 어려웠습니다.
  • 해결 방안 : AccessDeniedHandler를 커스텀 구현하여 인증 및 인가 실패 시의 동작을 정의했습니다. 이를 통해 클라이언트에 일관된 형식의 에러 응답을 제공하고, 프론트엔드에서 더 쉽게 에러 처리를 할 수 있게 되었습니다.
